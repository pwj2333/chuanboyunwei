# 🚢 船舶运维事件督办系统 - 本地开发使用指南

## 📦 前置要求

- Node.js 18+ （[下载地址](https://nodejs.org/)）
- 浏览器（推荐Chrome/Edge）

## 🚀 快速启动（一键启动）

### 方法一：双击启动脚本（推荐）

1. **双击 `启动.bat`**
   - 自动安装依赖
   - 自动初始化数据库
   - 自动启动前后端服务
   - 自动打开浏览器

2. **停止服务：双击 `停止.bat`**

### 方法二：手动启动

**后端启动：**
```bash
cd backend
npm install          # 首次需要
npm run init-db      # 首次需要，初始化数据库
npm start           # 启动服务
```

**前端启动（新开一个终端）：**
```bash
cd frontend
npm install          # 首次需要
npm run dev         # 启动服务
```

## 🔑 登录系统

- **访问地址**：http://localhost:5173
- **默认账号**：`admin`
- **默认密码**：`admin123`

## ⚙️ 首次使用配置

### 1. 配置AI（必需）

登录后进入"系统设置" → "AI配置"：

```
API Base URL: https://api.openai.com/v1
API Key: sk-xxxxx（你的OpenAI API Key）
模型名称: gpt-3.5-turbo
Temperature: 0.7
Max Tokens: 2000
```

点击"测试连接"确认配置正确，然后"保存配置"。

**支持的API：**
- OpenAI官方API
- 各种OpenAI兼容API（如：通义千问、智谱AI、Deepseek等）

### 2. 配置机器人通知（可选）

进入"系统设置" → "机器人通知"：

**企业微信机器人：**
1. 企业微信群聊 → 添加机器人 → 复制Webhook地址
2. 在系统中"添加机器人" → 选择"企业微信" → 粘贴URL

**飞书机器人：**
1. 飞书群聊 → 设置 → 机器人 → 添加机器人 → 复制Webhook
2. 在系统中"添加机器人" → 选择"飞书" → 粘贴URL

## 📖 基本使用流程

### 1. 添加船舶

看板页面 → 点击"添加船舶" → 输入船舶名称

### 2. 创建事件

看板页面 → 点击"新建事件"：
- 选择船舶
- 填写事件标题
- 填写描述（可选）
- 设置优先级（高/中/低）
- 设置督办时间（可选）

💡 创建时右侧会自动显示相似的历史事件供参考

### 3. 添加跟进记录

事件详情页 → 点击"添加记录"：
- 输入跟进内容
- 可上传附件（图片、文档，最多5个，单个最大10MB）

### 4. 修改事件状态

事件详情页 → 点击"修改状态"：
- 待处理
- 进行中
- 已完成

### 5. 归档事件并生成AI总结

事件完成后：
1. 将状态改为"已完成"
2. 点击"归档并生成AI总结"
3. 系统会调用AI分析整个事件过程
4. 生成结构化总结（摘要、问题类型、解决方案、经验教训）
5. 自动存入知识库

### 6. 查看知识库

- 创建新事件时，右侧自动显示相似历史事件
- 可以查看历史事件的AI总结
- 参考历史解决方案

## 👥 用户管理（管理员功能）

进入"用户管理"：
- 创建新用户
- 设置角色（管理员/普通用户）
- 修改用户信息
- 删除用户

**权限说明：**
- **管理员**：完整权限
- **普通用户**：查看和创建事件，不能删除事件和管理用户

## 🔔 定时提醒

系统每天9:00自动检查：
- 查找24小时内到期的事件
- 通过配置的机器人发送提醒
- 提醒内容包含：事件标题、船舶、优先级、到期时间

## ❓ 常见问题

### 1. 启动失败

**检查端口占用：**
```bash
# 检查3000端口（后端）
netstat -ano | findstr :3000

# 检查5173端口（前端）
netstat -ano | findstr :5173
```

**解决方案：**
- 关闭占用端口的程序
- 或修改端口：
  - 后端：修改 `backend/.env` 中的 `PORT=3000`
  - 前端：修改 `frontend/vite.config.js` 中的 `port: 5173`

### 2. AI总结失败

**原因：**
- AI配置不正确
- API Key无效或余额不足
- 网络无法访问API

**解决：**
1. 检查"系统设置"中的AI配置
2. 点击"测试连接"验证
3. 查看后端控制台日志

### 3. 文件上传失败

**检查：**
- 文件大小是否超过10MB
- 文件格式是否支持（图片、PDF、Office文档）
- `backend/uploads/` 目录是否有写入权限

### 4. 机器人通知不工作

**检查：**
1. Webhook URL是否正确
2. 机器人是否已启用
3. 点击"测试"按钮验证连接
4. 查看后端控制台日志
